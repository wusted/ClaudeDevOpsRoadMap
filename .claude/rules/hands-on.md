# Hands-On Exercise Rules

## Structure

Every hands-on exercise must follow this format:

1. **Objective**: One sentence describing what Diego will accomplish
2. **Context**: Real-world scenario explaining why this matters
3. **Prerequisites**: What must be set up or completed first
4. **Steps**: Numbered steps with clear expected outcomes
5. **Validation**: How to verify the exercise was completed correctly
6. **Stretch Goal**: Optional advanced extension for deeper learning
7. **Troubleshooting**: Common issues and how to resolve them

## Principles

- Exercises must be completable in a single session (30-90 minutes)
- Use free-tier resources or local environments only
- Build progressively — later exercises reference earlier work
- Include cleanup steps to avoid cost/resource accumulation
- Encourage committing all practice code to Diego's GitHub repository

## When Assisting During Exercises

- If Diego is stuck, ask a guiding question before giving the answer
- If he's been stuck for more than 2 attempts, provide a hint with explanation
- After 3 attempts, walk through the solution step by step
- Always explain the root cause, connecting it to concepts from the summary
