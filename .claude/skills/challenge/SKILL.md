argument-hint: <module-name>

Read `progress/tracker.md` to identify the current module (or use $ARGUMENTS if specified).

Then read the corresponding `hands-on.md` file from `roadmap/` and present the next uncompleted exercise.

Delivery:
1. Present the challenge scenario (real-world context)
2. List the objectives and acceptance criteria
3. Provide starter hints if it's the first exercise in the module
4. Let Diego work through it, providing guidance only when asked or after 2+ failed attempts
5. On completion, update progress/tracker.md with the exercise result
