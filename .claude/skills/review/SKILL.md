Read `progress/tracker.md` to identify completed modules.

If $ARGUMENTS is provided, review that specific module. Otherwise, review the most recently completed module.

Review format:
1. Ask 3-5 questions mixing conceptual and practical knowledge
2. Wait for Diego's answers
3. Provide feedback with corrections and deeper explanations where needed
4. If score is below 60%, suggest revisiting the module summary
5. If score is above 80%, confirm mastery and suggest moving forward

Question types to mix:
- "What would happen if..." (scenario-based)
- "How would you..." (practical application)
- "Why does X use Y instead of Z?" (design decisions)
- "Debug this..." (troubleshooting, plays to Diego's strength)
